# Ejercicios de práctica: `while True`, `break` y `continue`

## Problema 1: Contraseña correcta

Desarrolla un programa que solicite al usuario una contraseña.

- Si la contraseña es correcta, el programa debe finalizar.
- Si es incorrecta, debe volver a solicitarla.
- El programa debe continuar hasta que el usuario ingrese la contraseña correcta.

---

## Problema 2: Número positivo

Solicita al usuario un número.

- Si el número es positivo, el programa termina.
- Si es negativo o cero, debe volver a pedir otro número.

---

## Problema 3: Edad válida

Solicita la edad del usuario.

- La edad debe estar entre 1 y 120.
- Si el dato es inválido, vuelve a pedirlo.
- Cuando sea válido, finaliza el programa.

---

## Problema 4: Menú de salida

Muestra continuamente el siguiente menú:

1. Continuar
2. Salir

- Si el usuario selecciona "Salir", termina el programa.
- En cualquier otro caso, vuelve a mostrar el menú.

---

## Problema 5: Nombre obligatorio

Solicita el nombre del usuario.

- Si deja el campo vacío, vuelve a pedirlo.
- Cuando escriba un nombre válido, termina el programa.

---

## Problema 6: Número par

Solicita un número entero.

- Si el número es par, el programa finaliza.
- Si es impar, vuelve a solicitar otro número.

---

## Problema 7: Adivinar un número

El programa tiene un número secreto.

- El usuario debe intentar adivinarlo.
- Si falla, vuelve a intentarlo.
- Cuando acierte, el programa termina.

---

## Problema 8: Confirmación

Pregunta al usuario:

¿Desea continuar? (S/N)

- Si responde "S", continúa preguntando.
- Si responde "N", termina el programa.
- Si escribe cualquier otro valor, vuelve a preguntar.

---

## Problema 9: Múltiplo de 5

Solicita un número.

- Si el número es múltiplo de 5, termina el programa.
- Si no lo es, vuelve a pedir otro número.

---

## Problema 10: Nota aprobatoria

Solicita una nota.

- Si la nota es igual o superior a 4.0, termina.
- En caso contrario, vuelve a solicitar otra nota.

---

## Problema 11: Correo electrónico

Solicita un correo electrónico.

- Debe contener el carácter "@"
- Si no cumple la condición, vuelve a solicitarlo.
- Cuando sea válido, termina.
